Ocean Data Lab, 2022 Summer

My development product: view

Context

In the past, members of the Ocean Data Lab have collected over 1.5GB ocean acoustic data from large underwater networks. Various types of data visualizations are created to better analyze those data. However, Those visualizations requires different inputs such as location, time interval, graph type, etc, to be generated. It will be much convenient to manipulate these required parameters on an interface than writing Python codes.

Responsibility

Created a website where users can select different locations, graph types, and time interval to generate different data visualizations. Website also allows users to download the acoustic data in CSV file based on different requirements.

Takeaway

I chosed a tech combination (React + Flask) that I never tried before. My experience of building the interface highly improves my problem-solving skills and research skills. I got to know Professor Shima and a group of amazing researchers.

Final Project Prototype: The New Snake Game

This is my HCDE 438 reflection, where I reflect on my experience in the class and the projects I worked on.

Overview and Usage

Overview: I am using the p5.js library to create a gluttonous snake game. To start playing, click the "Start Game" button, and you will enter the game immediately. There is a score panel in the top left corner that displays the number of points you have earned (which corresponds to the number of red boxes the snake eats). If the snake eats itself or hits the wall (the edge of the browser window), the game is over. There are power-up foods embedded in this new version of the game. This special food will either speed up the snake or make the snake longer. You can restart the game by clicking the "Start Game" button again. Enjoy the game!

Or, you can click the linked text below to view the game page. https://andrewliu66.github.io/snake-game/

Here is the github repository link: https://github.com/AndrewLiu66/snake-game

Project scope
My project has become more fun and complex compared to the one I created for the tool chain assignment. The Snake game now has more interesting features that have been added to enhance the gameplay experience. One significant addition to the game is the second power-up ability, which required modifying the code to ensure that the snake could properly acquire both abilities. The two abilities are the blue box that appears on the screen, which gives the snake a new ability for a short amount of time, such as increasing speed or gaining 5 more points while getting longer. In addition, I have also added walls to the game, which increases the difficulty level. As a result, my project scope has expanded from its initial plan, but these additions have significantly improved the overall quality and user experience of the game.
Time management
During the development process of my Snake game prototype, although the new features may seem straightforward and simple, I definitely spent more time than I had initially planned. I encountered a few roadblocks that affected my original schedule. For instance, I had to spend more time than I anticipated on the power-up feature of the game. This involved modifying the existing code to ensure that the snake could acquire and use the two abilities properly. Additionally, I had to add walls to improve the game's difficulty. These unanticipated tasks added to the amount of time I spent building the prototype. Overall, my time management plan for building the prototype was good, but the unanticipated roadblocks and setbacks I encountered required me to be flexible and adjust my schedule accordingly.
Implementation plan
I would say that I am definitely on track to complete my final project. I have spent a decent amount of time ensuring that the core features of the updated Snake game work properly. This prototype represents a large portion of my final project, and for the final project, I will simply refine the style and instructions of the game to make it more engaging for users.
Development Process
Since this prototype was built on top of the old game, I will outline the steps that are new to the previous version. Here's a step-by-step overview of how to add new features to the gluttonous snake game using p5.js:
  1. Identify the new features to be added to the game.
  2. Research and gather necessary resources such as images, sounds, or libraries required to implement the new features.
  3. Create a list of tasks required to implement the new features.
  4. Update the HTML structure if necessary to accommodate the new features.
  5. Update the global variables and initialize them to default values.
  6. Modify the setup function to include new features such as power-ups, new game modes, or different difficulty levels.
  7. Modify the draw function to include new features such as displaying score, health, or a timer.
  8. Create new functions to handle the new features such as a function to spawn power-ups, a function to handle collisions with new game elements, or a function to change difficulty levels.
  9. Update the Snake class or any other class used in the game to accommodate the new features.
  10. Test the game with the new features and make necessary adjustments.
  11. Optimize the code and remove any bugs that may have arisen during the implementation of the new features.
  12. Publish the updated game for users to enjoy.

Issue
During the game creation process, I encountered an issue where the program could not assign the two power-up abilities to the snake properly. Here are my solutions:
  1. I located the function that defines the "if" statement that controls the type of ability the power-up fruit gives to the snake.
  2. I modified the condition in the if statement to fix the bug with the help of console.log.
  3. I replayed the game to ensure that it worked properly.
As a result of these steps, I was able to resolve the issue and make the game work.
