Reflection

This is my HCDE 438 reflection, where I reflect on my experience in the class and the projects I worked on.

Overview
I created a Google Chrome Extension to assist individuals in staying organized through the use of a to-do list. Utilizing HTML, CSS, and JavaScript, the extension is designed for ease of use in adding and deleting tasks. To witness its capabilities, simply open the "popup.html" file in your browser and the results will be displayed before your eyes. Here's a quick demonstration of the extension's functionality Here's a GIF of the extension in action: To use this extension, you will need to install it on your Google Chrome browser. You can do this by visiting the Chrome Web Store, searching for the extension and then adding it to your browser. Once it is installed, you will see the extension icon in the top right corner of your browser. Clicking on this icon will open the extension, and you can start using it to manage your tasks.

Development Process
  1. Planning: I started by outlining the functionality of the extension and creating a visual design for the layout. This included deciding on the overall layout, colors, and typography.
  2. Create Layout: I began by writing the HTML and CSS to create the basic layout of the extension. This included creating the structure of the page, and styling it to match the design.
  3. Make elements interactive: I then added the interactive elements, such as form fields and buttons, using JavaScript. This allowed users to add new tasks and remove existing tasks.
  4. Use Local Storage: To ensure that the user's data is saved even when they close the extension, I used Local Storage to store the data. This way, when the user opens the extension again, they will see their previous data.
  5. Testing and debugging: I tested the extension to make sure it doesn't throw errors when running on Google Chrome

Issue Deep-Dive
When making my to-do list using the Google Chrome Extension, I ran across an issue. The issue was that users' changes to the list when they added and removed things were not being preserved when the extension was closed and reopened. The list would always be empty when the extension was reopened since the user-entered data was not being preserved. After doing some investigation, I learned that the only way to save the user-entered data was to store and fetch data in the local storage. I then changed my code to store and retrieve data from local storage using the localStorage.getItem("tasks") method. This enabled me to store the user-entered information and make it accessible upon reopening the extension. I discovered the value of knowing how to save and retrieve data in local storage through this challenge when building a web application. I also discovered how crucial thorough testing and debugging are to finding and fixing code errors. I also learnt how crucial it is to do research, ask for assistance when necessary, and be knowledgeable about the developer tools the browser offers to help with testing and debugging.
